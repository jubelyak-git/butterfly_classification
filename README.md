# Japanese Butterfly Recognition & Similarity Search App
Веб-приложение на базе **Streamlit** и **PyTorch**, предназначенное для классификации 10 видов японских бабочек и поиска визуально похожих особей по загруженному изображению.

Главная особенность проекта — полностью **самостоятельно собранный датасет** и эффективная ML-архитектура, использующая векторные эмбеддинги для поиска визуально близких видов.

---

##  Основной функционал

* **Классификация видов:** Точное определение класса (вида) японской бабочки по фотографии.
* **Визуальный поиск (Similarity Search):** Отображение наиболее визуально похожих бабочек из базы данных на основе косинусного расстояния векторов.
* **Интерактивный UI:** Удобный веб-интерфейс на Streamlit, куда пользователь может загрузить любую фотографию для моментального анализа.

---

## Датасет (Dataset) & Борьба с дисбалансом

Датасет включает в себя **10 классов** японских бабочек (таких как *Сyrestis thyodamas*, *Hestina assimilis assimilis*, *Polygonia c-aureum* и др.) и был **собран и размечен вручную**.

### Решение проблемы дисбаланса классов (Imbalanced Data)
Так как датасет собирался вручную, количество фотографий для редких видов бабочек отличалось от распространенных. Для борьбы с дисбалансом классов в процессе обучения был применен метод **взвешенной функции потерь (Class Weighting)**:
1. С помощью `np.bincount` был подсчитан объем каждого класса в таргетах.
2. Были вычислены индивидуальные веса для классов (обратно пропорционально их частоте).
3. Веса были переданы напрямую в функцию потерь `nn.CrossEntropyLoss(weight=weights)`, что позволило модели одинаково хорошо штрафовать за ошибки как на популярных, так и на редких классах.

##  ML-архитектура и Извлечение Эмбеддингов

Был реализован красивый  подход **Feature Extraction**:

1.  **Fine-Tuning классификатора:** В  была взята предобученная модель **ResNet-50** из `torchvision.models`. Финальный полносвязный слой был адаптирован под наши задачи: `model.fc = nn.Linear(2048, 10)`. Модель была обучена в течение 20 эпох с динамическим изменением Learning Rate (разгон до Accuracy ~86% на валидации).
2.  **последнего слоя (Feature Extraction):** Для реализации Similarity Search мы отсекаем финальный классифицирующий слой `fc` (забираем эмбеддинги со слоя прямо перед ним, размерностью 2048).
3.  **Поиск сходства:** При загрузке новой фотографии Streamlit-приложение пропускает картинку через ResNet-50, извлекает вектор признаков (эмбеддинг) и находит топ ближайших бабочек из базы данных с помощью расчета **Cosine Similarity** (косинусного расстояния) между векторами.

## 💻 Визуальная часть (Streamlit Web App)

Интерфейс приложения написан на фреймворке **Streamlit**, что позволяет быстро разворачивать легковесные веб-сервисы для демонстрации работы моделей. Пользователь загружает фото -> получает предсказанный класс бабочки -> видит сетку похожих изображений из базы данных.
